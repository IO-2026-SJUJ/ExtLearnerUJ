sequenceDiagram
    actor U as Użytkownik
    participant UI as Interfejs Zgłoszenia
    participant S as System
    participant DB as Baza Danych
    participant NS as Serwis Powiadomień

    U->>UI: Wybiera opcję zgłoszenia
    UI-->>U: Wyświetla formularz zgłoszenia

    U->>UI: Określa typ zgłoszenia (materiał / użytkownik)
    UI-->>U: Dostosowuje formularz do typu zgłoszenia

    U->>UI: Podaje powód zgłoszenia i opis
    U->>UI: Zatwierdza zgłoszenie
    UI->>S: Przesyła dane zgłoszenia

    S->>S: Waliduje kompletność zgłoszenia
    S->>DB: Rejestruje zgłoszenie (typ, powód, autor, obiekt)
    DB-->>S: Potwierdzenie zapisu
    S->>DB: Ustawia status: do rozpatrzenia
    DB-->>S: Potwierdzenie

    S->>NS: Powiadom administratora o nowym zgłoszeniu
    NS-->>NS: Generuje powiadomienie
    NS-->>S: Powiadomienie wysłane

    S-->>UI: Zgłoszenie zarejestrowane
    UI-->>U: Wyświetla potwierdzenie wysłania zgłoszenia
